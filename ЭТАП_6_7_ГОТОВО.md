# АДМИНКА (Этап 6)

## 1. Дашборд

### GET /api/v1/admin/dashboard
**Получить статистику админки:**

**Возвращает:**
```json
{
  "users": {
    "total": 1500,
    "new_today": 25
  },
  "sellers": {
    "active": 120,
    "pending": 5
  },
  "orders": {
    "total": 5000,
    "today": 150
  },
  "revenue": {
    "total": 2500000.00,
    "today": 75000.00,
    "commission": 250000.00
  },
  "moderation": {
    "active_disputes": 3,
    "pending_withdrawals": 8
  }
}
```

**Что считается:**
- Всего пользователей и новых за сегодня
- Активные продавцы и заявки на модерацию
- Всего заказов и за сегодня
- Общая выручка, за сегодня, комиссия
- Активные споры и выводы на модерации

## 2. Управление пользователями

### GET /api/v1/admin/users
**Получить список пользователей:**

**Параметры:**
- search (optional): поиск по username или telegram_id
- is_blocked (optional): фильтр по статусу блокировки
- skip, limit: пагинация

### PATCH /api/v1/admin/users/{user_id}
**Обновить пользователя:**
```json
{
  "is_blocked": true,
  "balance_adjustment": 1000.00
}
```

**Логика:**
- Блокировка/разблокировка пользователя
- Корректировка баланса (+ или -)
- Создание транзакции при изменении баланса
- Запись в audit log

## 3. Управление продавцами

### GET /api/v1/admin/sellers
**Получить список продавцов:**

**Параметры:**
- status (optional): pending, active, suspended, banned
- skip, limit: пагинация

### PATCH /api/v1/admin/sellers/{seller_id}
**Обновить продавца:**
```json
{
  "status": "active",
  "is_verified": true
}
```

**Статусы:**
- pending → active (одобрить заявку)
- active → suspended (временная блокировка)
- active → banned (постоянная блокировка)

**Запись в audit log.**

## 4. Модерация лотов

### GET /api/v1/admin/lots
**Получить все лоты:**

**Параметры:**
- status (optional): draft, active, suspended, deleted
- seller_id (optional): фильтр по продавцу
- skip, limit: пагинация

### PATCH /api/v1/admin/lots/{lot_id}
**Модерировать лот:**
```json
{
  "status": "active",
  "rejection_reason": "Нарушение правил"
}
```

**Действия:**
- Одобрить (draft → active)
- Отклонить (draft → deleted)
- Заблокировать (active → suspended)

**Запись в audit log с причиной.**

## 5. Управление спорами

### GET /api/v1/admin/disputes
**Получить список споров:**

**Параметры:**
- status (optional): open, resolved
- skip, limit: пагинация

**Разрешение споров реализовано в `deals.py` (Этап 3):**
- POST /api/v1/deals/disputes/{dispute_id}/resolve

## 6. Управление выводами

### GET /api/v1/admin/withdrawals
**Получить список выводов:**

**Параметры:**
- status (optional): pending, completed, failed
- skip, limit: пагинация

### PATCH /api/v1/admin/withdrawals/{withdrawal_id}
**Обработать вывод:**
```json
{
  "status": "completed",
  "comment": "Выплачено на карту"
}
```

**Логика:**
- completed: вывод успешно обработан
- failed: вывод отклонён, средства возвращаются на баланс продавца

**Запись в audit log.**

## 7. Рассылки

### POST /api/v1/admin/broadcasts
**Создать рассылку:**
```json
{
  "message": "Важное объявление для всех пользователей!",
  "scheduled_at": "2024-01-20T10:00:00"
}
```

**Параметры:**
- message: текст сообщения (до 4096 символов)
- scheduled_at (optional): время отправки (если не указано - сразу)

**Статусы:**
- pending: ожидает отправки
- sending: отправляется
- sent: отправлено

### GET /api/v1/admin/broadcasts
**Получить историю рассылок:**

**Возвращает:**
- Список всех рассылок
- Статус, время отправки, количество получателей

## 8. Audit Logs

### GET /api/v1/admin/audit-logs
**Получить логи действий админов:**

**Параметры:**
- action_type (optional): фильтр по типу действия
- admin_id (optional): фильтр по админу
- skip, limit: пагинация

**Типы действий:**
- user_block, user_unblock
- balance_adjustment
- seller_status_change
- lot_moderation
- withdrawal_processed

**Каждый лог содержит:**
- admin_id: кто выполнил
- action_type: тип действия
- entity_type: тип сущности (user, seller, lot, etc.)
- entity_id: ID сущности
- description: описание
- metadata: дополнительные данные (JSON)
- created_at: время

# ОТЗЫВЫ (Этап 7)

## 1. Отзывы на товары

### GET /api/v1/products/{product_id}/reviews
**Получить отзывы на товар:**

**Возвращает:**
```json
{
  "items": [
    {
      "id": 1,
      "user_id": 123,
      "username": "user123",
      "rating": 5,
      "text": "Отличный товар!",
      "photos": ["url1", "url2"],
      "reply_text": "Спасибо за отзыв!",
      "created_at": "2024-01-15T10:00:00"
    }
  ],
  "total": 50,
  "average_rating": 4.8,
  "skip": 0,
  "limit": 20
}
```

**Только опубликованные отзывы (status = published).**

### POST /api/v1/orders/{order_id}/review
**Создать отзыв на товар:**
```json
{
  "rating": 5,
  "text": "Отличный товар, быстрая доставка!",
  "photos": ["url1", "url2", "url3"]
}
```

**Валидация:**
- rating: 1-5
- text: до 2000 символов (опционально)
- photos: до 5 фото (опционально)

**Проверки:**
- Заказ должен быть завершён
- Пользователь должен быть покупателем
- Нельзя оставить повторный отзыв

**Статус:** pending (ожидает модерации)

## 2. Отзывы на продавцов

### GET /api/v1/sellers/{seller_id}/reviews
**Получить отзывы на продавца:**

**Аналогично отзывам на товары.**

### POST /api/v1/deals/{deal_id}/review
**Создать отзыв на продавца:**
```json
{
  "rating": 5,
  "text": "Отличный продавец, всё быстро!"
}
```

**Валидация:**
- rating: 1-5
- text: до 2000 символов (опционально)

**Проверки:**
- Сделка должна быть завершена
- Пользователь должен быть покупателем
- Нельзя оставить повторный отзыв

**Статус:** pending (ожидает модерации)

## 3. Ответы на отзывы

### POST /api/v1/reviews/product/{review_id}/reply
**Продавец отвечает на отзыв о товаре:**
```json
{
  "reply_text": "Спасибо за отзыв! Рады что вам понравилось!"
}
```

**Проверки:**
- Только продавец может ответить
- Нельзя ответить дважды

### POST /api/v1/reviews/seller/{review_id}/reply
**Продавец отвечает на отзыв о себе:**

**Аналогично.**

## 4. Модерация отзывов (Админ)

### GET /api/v1/admin/reviews/pending
**Получить отзывы на модерации:**

**Параметры:**
- review_type: all, product, seller
- skip, limit: пагинация

**Возвращает все отзывы со статусом pending.**

### PATCH /api/v1/admin/reviews/product/{review_id}
**Модерировать отзыв на товар:**
```json
{
  "status": "published",
  "rejection_reason": "Нарушение правил"
}
```

**Статусы:**
- published: одобрить
- rejected: отклонить

**При публикации:**
- Пересчитывается средний рейтинг товара
- Обновляется product.rating

### PATCH /api/v1/admin/reviews/seller/{review_id}
**Модерировать отзыв на продавца:**

**Аналогично, но обновляется seller.rating.**

## 5. Расчёт рейтингов

**Автоматический пересчёт при публикации отзыва:**

**Для товаров:**
```sql
AVG(rating) WHERE product_id = X AND status = 'published'
```

**Для продавцов:**
```sql
AVG(rating) WHERE seller_id = X AND status = 'published'
```

**Округление до 2 знаков после запятой.**

## 🔐 Безопасность

**Админка:**
- Все эндпоинты требуют JWT + is_admin = True
- Проверка через `get_current_admin` dependency
- Все действия логируются в audit_logs

**Отзывы:**
- Только покупатели могут оставлять отзывы
- Только на завершённые заказы/сделки
- Нельзя оставить повторный отзыв
- Только продавцы могут отвечать на свои отзывы
- Модерация только для админов

## 🧪 Как тестировать

### Админка

#### 1. Получить дашборд
```bash
curl -H "Authorization: Bearer ADMIN_TOKEN" \
  http://localhost:8000/api/v1/admin/dashboard
```

#### 2. Заблокировать пользователя
```bash
curl -X PATCH http://localhost:8000/api/v1/admin/users/123 \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"is_blocked": true}'
```

#### 3. Одобрить продавца
```bash
curl -X PATCH http://localhost:8000/api/v1/admin/sellers/5 \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "active", "is_verified": true}'
```

#### 4. Обработать вывод
```bash
curl -X PATCH http://localhost:8000/api/v1/admin/withdrawals/10 \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "completed", "comment": "Выплачено"}'
```

#### 5. Создать рассылку
```bash
curl -X POST http://localhost:8000/api/v1/admin/broadcasts \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Важное объявление!",
    "scheduled_at": "2024-01-20T10:00:00"
  }'
```
