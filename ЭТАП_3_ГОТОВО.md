# Этап 3: Эскроу и сделки - ГОТОВО ✅

## ✅ Что реализовано

### Создан `api/routers/deals.py`

**Полная логика сделок с эскроу:**

### 1. GET /api/v1/deals/{deal_id} - Получить детали сделки
- Показывает статус сделки
- Информацию об эскроу
- Дату автозавершения
- Подтверждение покупателя

### 2. POST /api/v1/deals/{deal_id}/deliver - Доставка товара (продавец)
- Продавец отправляет товар
- Обновляет статус на WAITING_CONFIRMATION
- Отправляет уведомление покупателю
- Запускает таймер автозавершения (72 часа)

### 3. POST /api/v1/deals/{deal_id}/confirm - Подтверждение доставки (покупатель)
- Покупатель подтверждает получение
- Обновляет статус на COMPLETED
- Освобождает эскроу продавцу
- Создаёт транзакции

### 4. POST /api/v1/deals/{deal_id}/dispute - Открыть спор
- Покупатель или продавец открывает спор
- Обновляет статус на DISPUTE
- Останавливает автозавершение
- Уведомляет админа

### 5. GET /api/v1/deals/{deal_id}/dispute - Получить детали спора
- Показывает причину спора
- Описание проблемы
- Статус разрешения
- Комментарий админа

### 6. POST /api/v1/deals/disputes/{dispute_id}/resolve - Разрешить спор (админ)
**3 варианта разрешения:**

**a) release_to_seller** - Отдать продавцу
- Освобождает эскроу продавцу
- Завершает сделку
- Создаёт транзакции

**b) refund_to_buyer** - Вернуть покупателю
- Возвращает полную сумму покупателю
- Обновляет статус на REFUNDED
- Создаёт транзакцию возврата

**c) partial** - Частичный возврат
- Возвращает часть покупателю
- Остаток отдаёт продавцу (минус комиссия)
- Завершает сделку
- Создаёт транзакции для обоих

### 7. POST /api/v1/deals/auto-complete - Автозавершение (cron)
- Находит сделки с истёкшим таймером
- Автоматически завершает их
- Освобождает эскроу продавцу
- Помечает как автозавершённые

## 🔧 Внутренние функции

### _release_escrow(deal, session)
**Освобождение эскроу продавцу:**
- Получает seller и user
- Добавляет seller_amount на баланс
- Создаёт Transaction(SALE) для продавца
- Создаёт Transaction(COMMISSION) для комиссии
- Помечает escrow_released = True

### _refund_to_buyer(deal, session)
**Полный возврат покупателю:**
- Получает buyer
- Добавляет полную сумму на баланс
- Создаёт Transaction(REFUND)

### _partial_refund(deal, refund_amount, session)
**Частичный возврат:**
- Возвращает refund_amount покупателю
- Рассчитывает остаток для продавца
- Вычитает комиссию
- Создаёт транзакции для обоих
- Помечает escrow_released = True

## 📊 Статусы сделок

```
CREATED → PAID → IN_PROGRESS → WAITING_CONFIRMATION → COMPLETED
                      ↓
                  DISPUTE → (admin resolves) → COMPLETED / REFUNDED
```

**Статусы:**
- `CREATED` - Сделка создана
- `PAID` - Оплачена, средства в эскроу
- `IN_PROGRESS` - Продавец работает над доставкой
- `WAITING_CONFIRMATION` - Ожидает подтверждения от покупателя
- `COMPLETED` - Завершена, эскроу освобождён
- `DISPUTE` - Открыт спор
- `REFUNDED` - Возврат средств
- `CANCELED` - Отменена

## 🕐 Автозавершение

**Логика:**
1. При создании deal устанавливается `auto_complete_at = now() + 72 hours`
2. Cron job каждый час вызывает `/api/v1/deals/auto-complete`
3. Находит сделки где `auto_complete_at <= now()` и `status = WAITING_CONFIRMATION`
4. Автоматически завершает их
5. Освобождает эскроу продавцу

**Настройка cron:**
```bash
# Каждый час
0 * * * * curl -X POST http://localhost:8000/api/v1/deals/auto-complete
```

## 🧪 Как тестировать

### 1. Получить детали сделки
```bash
curl http://localhost:8000/api/v1/deals/1
```

### 2. Продавец доставляет товар
```bash
curl -X POST http://localhost:8000/api/v1/deals/1/deliver \
  -H "Content-Type: application/json" \
  -d '{
    "delivery_data": "Код активации: ABC-123-XYZ",
    "comment": "Товар отправлен"
  }'
```

### 3. Покупатель подтверждает доставку
```bash
curl -X POST http://localhost:8000/api/v1/deals/1/confirm \
  -H "Content-Type: application/json" \
  -d '{
    "rating": 5,
    "comment": "Всё отлично!"
  }'
```

### 4. Открыть спор
```bash
curl -X POST http://localhost:8000/api/v1/deals/1/dispute \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "not_received",
    "description": "Товар не получен, продавец не отвечает"
  }'
```

### 5. Получить детали спора
```bash
curl http://localhost:8000/api/v1/deals/1/dispute
```

### 6. Админ разрешает спор (возврат покупателю)
```bash
curl -X POST http://localhost:8000/api/v1/deals/disputes/1/resolve \
  -H "Content-Type: application/json" \
  -d '{
    "resolution": "refund_to_buyer",
    "comment": "Продавец не предоставил доказательства доставки"
  }'
```

### 7. Админ разрешает спор (частичный возврат)
```bash
curl -X POST http://localhost:8000/api/v1/deals/disputes/1/resolve \
  -H "Content-Type: application/json" \
  -d '{
    "resolution": "partial",
    "partial_amount": 50.00,
    "comment": "Товар доставлен частично"
  }'
```

### 8. Запустить автозавершение
```bash
curl -X POST http://localhost:8000/api/v1/deals/auto-complete
```

## 🔐 Безопасность

**Проверки доступа:**
- Только покупатель и продавец видят детали сделки
- Только продавец может доставить товар
- Только покупатель может подтвердить доставку
- Только участники могут открыть спор
- Только админ может разрешить спор

**Проверки статусов:**
- Нельзя доставить товар для завершённой сделки
- Нельзя подтвердить доставку если товар не доставлен
- Нельзя открыть спор для завершённой сделки
- Нельзя открыть второй спор для одной сделки
