# Функции маркетплейса - Краткое описание

## ✅ Реализовано (Фаза 1)

### 1. Система продавцов
- Регистрация пользователей как продавцов
- Профиль продавца с рейтингом и статистикой
- Верификация продавцов администратором
- Баланс продавца для получения оплат
- Комиссия платформы (настраивается индивидуально)

**Команды:**
- `/seller` - меню продавца
- Создание магазина через бота
- Просмотр статистики продаж

### 2. Лоты (Предложения)
- Продавцы создают лоты для товаров
- Несколько продавцов могут продавать один товар
- Разные цены от разных продавцов
- Управление остатками товара
- Статусы лотов (черновик, активный, приостановлен, нет в наличии)

**Типы доставки:**
- ⚡ Автоматическая - мгновенная выдача кодов
- 👤 Ручная - продавец отправляет товар вручную
- 📍 По координатам - для игровых предметов

### 3. Автовыдача товаров
- Загрузка кодов/данных для автоматической выдачи
- Резервирование товара при покупке
- Автоматическая доставка после оплаты
- Управление остатками
- Автоматическое обновление статуса "нет в наличии"

### 4. Сделки (Deals)
- Создание сделки при каждой покупке
- Связь покупателя и продавца
- Отслеживание статуса сделки
- Автоматическое завершение через 3 дня
- История всех сделок

**Статусы сделки:**
- 🆕 Создана
- 💳 Оплачена
- ⏳ В процессе
- ⏰ Ожидает подтверждения
- ✅ Завершена
- ❌ Отменена
- ⚠️ Спор
- 💸 Возврат

### 5. Чат покупатель-продавец
- Личные сообщения в рамках сделки
- Отправка текста и медиа
- Системные сообщения о статусе
- Уведомления о новых сообщениях
- Отметка прочитанных сообщений

### 6. Балансы и транзакции
- Баланс покупателя для оплаты
- Баланс продавца для получения средств
- Полная история всех транзакций
- Типы транзакций:
  - 💰 Пополнение
  - 🛒 Покупка
  - 💼 Продажа
  - 💸 Возврат
  - 📤 Вывод средств
  - 💵 Комиссия
  - 🎁 Бонус
  - ⚠️ Штраф

### 7. Гарант-сервис (Эскроу)
- Заморозка средств при покупке
- Средства хранятся до подтверждения
- Автоматическое завершение через 3 дня
- Защита покупателя и продавца
- Возврат при отмене

### 8. Отзывы о продавцах
- Отзывы после завершения сделки
- Рейтинг от 1 до 5 звезд
- Текстовые отзывы
- Ответы продавцов на отзывы
- Автоматический пересчет рейтинга

### 9. Уведомления
- Новые сообщения в сделке
- Новые заказы для продавца
- Изменение статуса заказа
- Получение оплаты
- Новые отзывы
- Системные уведомления

### 10. Избранное
- Добавление лотов в избранное
- Быстрый доступ к понравившимся товарам
- Уведомления о снижении цены (будущая функция)

### 11. Споры и арбитраж
- Открытие спора покупателем или продавцом
- Рассмотрение администратором
- Решение в пользу одной из сторон
- Возврат средств или завершение сделки
- История споров

### 12. Вывод средств
- Заявки на вывод средств продавцами
- Минимальная сумма вывода
- Обработка администратором
- Статусы заявок
- История выводов

## 🎯 Как это работает

### Сценарий покупки (автовыдача):
1. Покупатель выбирает товар
2. Видит список лотов от разных продавцов
3. Выбирает лот (по цене, рейтингу продавца)
4. Оплачивает с баланса
5. Создается сделка
6. Товар резервируется
7. Код автоматически доставляется покупателю
8. Покупатель подтверждает получение (или автоматически через 3 дня)
9. Средства переводятся продавцу (минус комиссия)
10. Сделка завершена

### Сценарий покупки (ручная доставка):
1-5. Аналогично автовыдаче
6. Продавец получает уведомление о заказе
7. Открывается чат между покупателем и продавцом
8. Продавец отправляет товар через чат
9. Покупатель подтверждает получение
10. Средства переводятся продавцу
11. Сделка завершена

### Сценарий продажи:
1. Пользователь регистрируется как продавец (`/seller`)
2. Администратор активирует продавца
3. Продавец создает лот для товара
4. Для автовыдачи - загружает коды
5. Активирует лот
6. Лот появляется в каталоге
7. При покупке - получает уведомление
8. Для ручной доставки - общается с покупателем
9. После завершения - получает оплату на баланс
10. Может вывести средства

## 🔧 Технические детали

### Архитектура:
- **Repositories** - работа с БД
- **Services** - бизнес-логика
- **Handlers** - обработка команд бота
- **Keyboards** - клавиатуры
- **States** - FSM состояния

### Новые сервисы:
- `SellerService` - управление продавцами
- `LotService` - управление лотами
- `DealService` - управление сделками

### Новые хендлеры:
- `seller.py` - функции продавца
- `deals.py` - управление сделками

### Автоматические задачи:
- `auto_complete_deals.py` - автозавершение сделок
- `cleanup_reservations.py` - освобождение резерваций
