# Этап 4: WebSocket чат - ГОТОВО ✅

## ✅ Что реализовано

### Создан `api/routers/chat.py`

**Полная система real-time чата между покупателем и продавцом:**

### 1. WebSocket /ws/chat/{deal_id} - Real-time чат
**Аутентификация:**
- Query параметр `token` (JWT access token)
- Проверка что пользователь - участник сделки (buyer или seller)
- Автоматическое закрытие соединения при неверном токене

**Типы сообщений:**

#### a) text - Текстовое сообщение
```json
{
  "type": "text",
  "content": "Привет! Когда отправите товар?"
}
```
- Сохраняется в БД (таблица `deal_messages`)
- Отправляется обеим сторонам
- Возвращает подтверждение с `message_id`

#### b) typing - Индикатор печати
```json
{
  "type": "typing",
  "is_typing": true
}
```
- Показывает что пользователь печатает
- Отправляется только другой стороне
- Не сохраняется в БД

#### c) read - Прочитать сообщение
```json
{
  "type": "read",
  "message_id": 123
}
```
- Помечает сообщение как прочитанное
- Обновляет `is_read = True` и `read_at`
- Отправляет `read_receipt` отправителю

#### d) read_all - Прочитать все
```json
{
  "type": "read_all"
}
```
- Помечает все непрочитанные сообщения как прочитанные
- Отправляет `read_all_receipt` другой стороне

#### e) ping - Heartbeat
```json
{
  "type": "ping"
}
```
- Проверка соединения
- Возвращает `{"type": "pong"}`

### 2. GET /api/v1/deals/{deal_id}/messages - История сообщений
**Параметры:**
- `skip` - пропустить N сообщений (default: 0)
- `limit` - максимум сообщений (default: 50)

**Возвращает:**
- Список сообщений в обратном порядке (новые первыми)
- Для каждого: id, sender_id, content, is_read, read_at, created_at

### 3. GET /api/v1/deals/{deal_id}/unread-count - Счётчик непрочитанных
**Возвращает:**
- Количество непрочитанных сообщений от другой стороны

## 🔧 ConnectionManager

**Управление WebSocket соединениями:**

### Структура данных:
```python
active_connections: Dict[int, Dict[int, WebSocket]]
# deal_id -> {user_id -> websocket}

user_connections: Dict[int, Set[int]]
# user_id -> set of deal_ids
```

### Методы:

#### connect(websocket, deal_id, user_id)
- Принимает WebSocket соединение
- Добавляет в active_connections
- Отправляет подтверждение подключения

#### disconnect(deal_id, user_id)
- Удаляет соединение
- Очищает пустые записи

#### send_personal_message(message, deal_id, user_id)
- Отправляет сообщение конкретному пользователю
- Обрабатывает ошибки отправки

#### broadcast_to_deal(message, deal_id, exclude_user)
- Отправляет сообщение всем участникам сделки
- Может исключить отправителя

## 🔐 Безопасность

**Проверки доступа:**
- JWT токен обязателен для подключения
- Только buyer и seller могут подключиться к чату сделки
- Нельзя прочитать чужие сообщения
- Нельзя пометить свои сообщения как прочитанные

**Обработка ошибок:**
- Автоматическое отключение при ошибке аутентификации
- Graceful disconnect при потере соединения
- Логирование всех ошибок

## 🎯 Reconnection Logic (для Frontend)

**Рекомендуемая стратегия переподключения:**

```javascript
class ChatWebSocket {
  constructor(dealId, token) {
    this.dealId = dealId;
    this.token = token;
    this.ws = null;
    this.reconnectAttempts = 0;
    this.maxReconnectAttempts = 5;
    this.reconnectDelay = 1000; // 1 секунда
    this.messageQueue = [];
    this.connect();
  }

  connect() {
    this.ws = new WebSocket(
      `ws://localhost:8000/api/v1/ws/chat/${this.dealId}?token=${this.token}`
    );

    this.ws.onopen = () => {
      console.log("Connected to chat");
      this.reconnectAttempts = 0;
      this.reconnectDelay = 1000;
      
      // Отправить очередь сообщений
      while (this.messageQueue.length > 0) {
        const msg = this.messageQueue.shift();
        this.ws.send(JSON.stringify(msg));
      }
    };

    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      this.handleMessage(data);
    };

    this.ws.onerror = (error) => {
      console.error("WebSocket error:", error);
    };

    this.ws.onclose = () => {
      console.log("Disconnected from chat");
      this.reconnect();
    };
  }

  reconnect() {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) {
      console.error("Max reconnect attempts reached");
      return;
    }

    this.reconnectAttempts++;
    const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts - 1);
    
    console.log(`Reconnecting in ${delay}ms (attempt ${this.reconnectAttempts})`);
    
    setTimeout(() => {
      this.connect();
    }, delay);
  }

  send(message) {
    if (this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(message));
    } else {
      // Добавить в очередь
      this.messageQueue.push(message);
    }
  }

  handleMessage(data) {
    // Обработка входящих сообщений
    console.log("Received:", data);
  }

  close() {
    this.reconnectAttempts = this.maxReconnectAttempts; // Отключить переподключение
    this.ws.close();
  }
}
```
