# 🔍 Что проверить в Backend

## Быстрая проверка (5 минут)

### 1. Запустить сервер
```bash
cd ПРОЕКТЫ/gpay-main
uvicorn api.main:app --reload
```

**Ожидаемый результат:**
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     🚀 Starting FastAPI application...
```

### 2. Проверить health check
```bash
curl http://localhost:8000/health
```

**Ожидаемый результат:**
```json
{"status": "healthy", "service": "p2p-marketplace-api"}
```

### 3. Открыть Swagger UI
Перейти в браузере: http://localhost:8000/api/docs

**Должны увидеть:**
- 12 разделов (Authentication, Users, Catalog, Cart, Orders, Deals, Chat, Sellers, Reviews, Admin, Payments, Notifications)
- 80+ эндпоинтов

## Полная проверка (30 минут)

### 1. Аутентификация
```bash
# Создать тестового пользователя (нужен валидный Telegram initData)
curl -X POST http://localhost:8000/api/v1/auth/telegram \
  -H "Content-Type: application/json" \
  -d '{"init_data": "YOUR_INIT_DATA"}'
```

**Получите JWT токен для дальнейших запросов.**

### 2. Каталог
```bash
# Получить игры
curl http://localhost:8000/api/v1/games

# Получить категории
curl http://localhost:8000/api/v1/categories

# Получить лоты с фильтрами
curl "http://localhost:8000/api/v1/lots?game_id=1&sort=price_asc&limit=10"
```

### 3. Корзина
```bash
# Добавить в корзину
curl -X POST http://localhost:8000/api/v1/cart/items \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "lot_id": 1,
    "quantity": 1
  }'

# Получить корзину
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:8000/api/v1/cart
```

### 4. Заказ
```bash
# Создать заказ
curl -X POST http://localhost:8000/api/v1/orders \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "idempotency_key": "unique-key-123"
  }'
```

### 5. WebSocket чат
**Подключиться через JavaScript:**
```javascript
const ws = new WebSocket('ws://localhost:8000/api/v1/ws/chat/1?token=YOUR_TOKEN');

ws.onopen = () => {
  console.log('Connected');
  ws.send(JSON.stringify({
    type: 'text',
    content: 'Привет!'
  }));
};

ws.onmessage = (event) => {
  console.log('Received:', JSON.parse(event.data));
};
```

### 6. Продавец
```bash
# Подать заявку на продавца
curl -X POST http://localhost:8000/api/v1/sellers/apply \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "shop_name": "Мой магазин",
    "description": "Продаю игровые товары"
  }'

# Получить дашборд (после одобрения админом)
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:8000/api/v1/sellers/me/dashboard
```

### 7. Админка
```bash
# Получить дашборд (нужен токен админа)
curl -H "Authorization: Bearer ADMIN_TOKEN" \
  http://localhost:8000/api/v1/admin/dashboard

# Одобрить продавца
curl -X PATCH http://localhost:8000/api/v1/admin/sellers/1 \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "active",
    "is_verified": true
  }'
```

### 8. Отзывы
```bash
# Оставить отзыв на товар (после завершения заказа)
curl -X POST http://localhost:8000/api/v1/orders/1/review \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "rating": 5,
    "text": "Отличный товар!"
  }'

# Получить отзывы на товар
curl http://localhost:8000/api/v1/products/1/reviews
```

## Проверка безопасности

```bash
curl http://localhost:8000/api/v1/users/me
```

**Ожидаемый результат:**
```json
{"detail": "Not authenticated"}
```

### 2. Попытка доступа к админке без прав
```bash
curl -H "Authorization: Bearer USER_TOKEN" \
  http://localhost:8000/api/v1/admin/dashboard
```

**Ожидаемый результат:**

## Проверка производительности

### 1. Проверить время ответа API
```bash
time curl http://localhost:8000/api/v1/games
```

**Ожидаемый результат:** < 200ms

### 2. Проверить кеширование
```bash
# Первый запрос (без кеша)
time curl http://localhost:8000/api/v1/games

# Второй запрос (с кешем)
time curl http://localhost:8000/api/v1/games
```

**Второй запрос должен быть быстрее.**

### 3. Нагрузочное тестирование (опционально)
```bash
# Установить Apache Bench
apt-get install apache2-utils

# 1000 запросов, 10 одновременно
ab -n 1000 -c 10 http://localhost:8000/api/v1/games
```

```bash
  -H "Content-Type: application/json" \
  -d '{
    "type": "notification",
    "event": "payment.succeeded",
    "object": {
      "id": "test-payment-id",
      "status": "succeeded",
      "amount": {"value": "500.00", "currency": "RUB"},
      "metadata": {"payment_id": "1"}
    }
  }'
```

**Проверить что:**
- Платёж обновился в БД
- Заказ перешёл в статус paid
- Создалась сделка
