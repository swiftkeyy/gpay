# Этап 5: Личный кабинет продавца - ГОТОВО ✅

## ✅ Что реализовано

### Обновлён `api/routers/sellers.py`

**Полный функционал личного кабинета продавца:**

## 1. Заявка на продавца

### POST /api/v1/sellers/apply
**Подать заявку на статус продавца:**
```json
{
  "shop_name": "Мой магазин",
  "description": "Продаю игровые товары"
}
```

**Валидация:**
- shop_name: 3-120 символов
- description: до 2000 символов (опционально)

**Проверки:**
- Нельзя подать повторную заявку если уже pending
- Нельзя подать заявку если уже продавец

**Возвращает:**
- seller_id
- status: "pending"

## 2. Профиль продавца

### GET /api/v1/sellers/me
**Получить профиль продавца:**
```json
{
  "id": 1,
  "shop_name": "Мой магазин",
  "description": "Продаю игровые товары",
  "status": "active",
  "is_verified": true,
  "rating": 4.8,
  "total_sales": 150,
  "balance": 12500.00,
  "created_at": "2024-01-01T00:00:00"
}
```

### PATCH /api/v1/sellers/me
**Обновить профиль:**
```json
{
  "shop_name": "Новое название",
  "description": "Новое описание"
}
```

## 3. Дашборд продавца

### GET /api/v1/sellers/me/dashboard
**Получить статистику продаж:**

**Возвращает:**
```json
{
  "balance": {
    "available": 12500.00,
    "pending_withdrawals": 5000.00,
    "in_escrow": 0.00
  },
  "today": {
    "sales": 5,
    "revenue": 2500.00
  },
  "week": {
    "sales": 35,
    "revenue": 17500.00
  },
  "month": {
    "sales": 120,
    "revenue": 60000.00
  },
  "all_time": {
    "sales": 500,
    "revenue": 250000.00
  },
  "performance": {
    "rating": 4.8,
    "total_sales": 500,
    "active_deals": 3
  }
}
```

**Что считается:**
- **today**: продажи с начала текущего дня (00:00)
- **week**: продажи за последние 7 дней
- **month**: продажи за последние 30 дней
- **all_time**: все завершённые сделки
- **active_deals**: сделки в статусе paid/in_progress/waiting_confirmation
- **pending_withdrawals**: сумма выводов в статусе pending

## 4. Управление лотами

### GET /api/v1/sellers/me/lots
**Получить список лотов:**

**Параметры:**
- status (optional): draft, active, out_of_stock, suspended, deleted
- skip (default: 0)
- limit (default: 20)

**Возвращает:**
```json
{
  "items": [
    {
      "id": 1,
      "title": "Аккаунт Brawl Stars",
      "price": 500.00,
      "status": "active",
      "delivery_type": "auto",
      "views": 150,
      "sales": 25,
      "created_at": "2024-01-01T00:00:00"
    }
  ],
  "total": 1,
  "skip": 0,
  "limit": 20
}
```

### POST /api/v1/sellers/me/lots
**Создать новый лот:**
```json
{
  "title": "Аккаунт Brawl Stars",
  "description": "Полное описание товара",
  "price": 500.00,
  "game_id": 1,
  "category_id": 2,
  "product_id": 3,
  "delivery_type": "auto",
  "stock_quantity": 10
}
```

**Валидация:**
- title: 3-255 символов
- description: до 5000 символов
- price: положительное число, 2 знака после запятой
- delivery_type: auto, instant, manual

**Проверки:**
- Продавец должен быть в статусе "active"

**Возвращает:**
- lot_id
- status: "draft"

### PATCH /api/v1/sellers/me/lots/{lot_id}
**Обновить лот:**
```json
{
  "title": "Новое название",
  "description": "Новое описание",
  "price": 600.00,
  "delivery_type": "manual",
  "status": "active"
}
```

**Все поля опциональны.**

### DELETE /api/v1/sellers/me/lots/{lot_id}
**Удалить лот (soft delete):**
- Устанавливает status = "deleted"
- Лот не удаляется из БД

## 5. Управление стоком (для auto-delivery)

### POST /api/v1/sellers/me/lots/{lot_id}/stock
**Добавить товары в сток:**
```json
[
  {"data": "login:password1"},
  {"data": "login:password2"},
  {"data": "login:password3"}
]
```

**Проверки:**
- Лот должен быть с delivery_type = "auto"
- Каждый data до 10000 символов

**Логика:**
- Создаёт записи в lot_stock_items
- Если лот был out_of_stock, меняет статус на active

## 6. Поднятие лотов (Boost)

### POST /api/v1/sellers/me/lots/{lot_id}/boost
**Поднять лот в топ поиска:**
```json
{
  "duration_hours": 24
}
```

**Параметры:**
- duration_hours: 1-168 (от 1 часа до 7 дней)

**Стоимость:**
- 10 RUB за час

**Логика:**
- Проверяет баланс продавца
- Списывает стоимость с баланса
- Устанавливает boosted_until = now + duration_hours
- Создаёт транзакцию типа "boost"

**Возвращает:**
```json
{
  "message": "Lot boosted successfully",
  "boosted_until": "2024-01-16T10:30:00",
  "cost": 240.00,
  "new_balance": 12260.00
}
```

## 7. Вывод средств

### POST /api/v1/sellers/me/withdrawals
**Запросить вывод средств:**
```json
{
  "amount": 5000.00,
  "payment_method": "card",
  "payment_details": "4111111111111111"
}
```

**Параметры:**
- amount: положительное число, 2 знака после запятой
- payment_method: card, qiwi, yoomoney, crypto
- payment_details: до 500 символов

**Проверки:**
- Баланс >= amount
- Минимальная сумма: 100 RUB

**Логика:**
- Списывает amount с баланса продавца
- Создаёт запись в withdrawals со статусом "pending"
- Админ обрабатывает вручную

**Возвращает:**
```json
{
  "withdrawal_id": 1,
  "amount": 5000.00,
  "status": "pending",
  "message": "Withdrawal request submitted"
}
```

### GET /api/v1/sellers/me/withdrawals
**Получить историю выводов:**

**Параметры:**
- skip (default: 0)
- limit (default: 20)

**Возвращает:**
```json
{
  "items": [
    {
      "id": 1,
      "amount": 5000.00,
      "payment_method": "card",
      "status": "pending",
      "created_at": "2024-01-15T10:00:00",
      "processed_at": null
    }
  ],
  "total": 1,
  "skip": 0,
  "limit": 20
}
```

## 🔐 Безопасность

**Проверки доступа:**
- Все эндпоинты требуют JWT аутентификацию
- Только владелец может управлять своими лотами
- Только активные продавцы могут создавать лоты
- Нельзя вывести больше чем на балансе

**Валидация:**
- Все строки проверяются на длину
- Цены проверяются на положительность
- Enum поля проверяются на допустимые значения
